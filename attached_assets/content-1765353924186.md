## Tab

| JSON 표현 |
| --- |
| ```<br>{<br>  "tabProperties": {<br>    object (TabProperties)<br>  },<br>  "childTabs": [<br>    {<br>      object (Tab)<br>    }<br>  ],<br>  // Union field content can be only one of the following:<br>  "documentTab": {<br>    object (DocumentTab)<br>  }<br>  // End of list of possible types for union field content.<br>}<br>``` |

| 필드 |
| --- |
| `tabProperties` | `object (TabProperties)`<br>탭의 속성(예: ID, 제목)입니다. |

## TabProperties

탭의 속성입니다.

| JSON 표현 |
| --- |
| ```<br>{<br>  "tabId": string,<br>  "title": string,<br>  "parentTabId": string,<br>  "index": integer,<br>  "nestingLevel": integer,<br>  "iconEmoji": string<br>}<br>``` |

| 필드 |
| --- |
| `tabId` | `string`<br>출력 전용입니다. 탭의 ID입니다. 이 필드는 변경할 수 없습니다. |
| `title` | `string`<br>사용자에게 표시되는 탭 이름입니다. |
| `parentTabId` | `string`<br>선택사항입니다. 상위 탭의 ID입니다. 현재 탭이 루트 수준 탭인 경우 비어 있습니다. 즉, 상위 탭이 없습니다. |
| `index` | `integer`<br>상위 요소 내 탭의 0부터 시작하는 색인입니다. |
| `nestingLevel` | `integer`<br>출력 전용입니다. 문서 내 탭의 깊이입니다. 루트 수준 탭은 0부터 시작합니다. |
| `iconEmoji` | `string`<br>선택사항입니다. 탭과 함께 표시되는 그림 이모티콘 아이콘 유효한 이모티콘 아이콘은 비어 있지 않은 유니코드 문자열로 표현됩니다. 단일 그림 이모티콘을 나타내지 않는 문자 집합은 잘못된 것입니다.<br>이모티콘이 유효하지 않으면 400 잘못된 요청 오류가 반환됩니다.<br>이 값이 설정되지 않았거나 비어 있으면 탭에 기본 탭 아이콘이 표시됩니다. |

## StructuralElement

StructuralElement는 문서에 구조를 제공하는 콘텐츠를 설명합니다.

| JSON 표현 |
| --- |
| ```<br>{<br>  "startIndex": integer,<br>  "endIndex": integer,<br>  // Union field content can be only one of the following:<br>  "paragraph": {<br>    object (Paragraph)<br>  },<br>  "sectionBreak": {<br>    object (SectionBreak)<br>  },<br>  "table": {<br>    object (Table)<br>  },<br>  "tableOfContents": {<br>    object (TableOfContents)<br>  }<br>  // End of list of possible types for union field content.<br>}<br>``` |

| 필드 |
| --- |
| `startIndex` | `integer`<br>이 구조적 요소의 0 기반 시작 색인(UTF-16 코드 단위)입니다. |
| `endIndex` | `integer`<br>이 구조적 요소의 0부터 시작하는 종료 색인입니다(제외). UTF-16 코드 단위로 표시됩니다. |
| 통합 필드 `content`. 구조적 요소의 콘텐츠입니다. `content`은 다음 중 하나여야 합니다. |
| `paragraph` | `object (Paragraph)`<br>단락 유형의 구조적 요소입니다. |
| `sectionBreak` | `object (SectionBreak)`<br>구조적 요소의 섹션 나누기 유형입니다. |
| `table` | `object (Table)`<br>구조적 요소의 표 유형입니다. |
| `tableOfContents` | `object (TableOfContents)`<br>목차 유형의 구조적 요소입니다. |

## 단락

단락을 나타내는 `StructuralElement`입니다. 단락은 줄바꿈 문자로 종료되는 콘텐츠 범위입니다.

| JSON 표현 |
| --- |
| ```<br>{<br>  "elements": [<br>    {<br>      object (ParagraphElement)<br>    }<br>  ],<br>  "paragraphStyle": {<br>    object (ParagraphStyle)<br>  },<br>  "suggestedParagraphStyleChanges": {<br>    string: {<br>      object (SuggestedParagraphStyle)<br>    },<br>    ...<br>  },<br>  "bullet": {<br>    object (Bullet)<br>  },<br>  "suggestedBulletChanges": {<br>    string: {<br>      object (SuggestedBullet)<br>    },<br>    ...<br>  },<br>  "positionedObjectIds": [<br>    string<br>  ],<br>  "suggestedPositionedObjectIds": {<br>    string: {<br>      object (ObjectReferences)<br>    },<br>    ...<br>  }<br>}<br>``` |

| 필드 |
| --- |
| `elements[]` | `object (ParagraphElement)`<br>단락의 콘텐츠를 구성요소로 나눈 것입니다. |
| `paragraphStyle` | `object (ParagraphStyle)`<br>이 단락의 스타일입니다. |
| `suggestedParagraphStyleChanges` | `map (key: string, value: object (SuggestedParagraphStyle))`<br>추천된 단락 스타일이 제안 ID로 키가 지정된 이 단락으로 변경됩니다. |

| `suggestedBulletChanges` | `map (key: string, value: object (SuggestedBullet))`<br>이 단락의 글머리기호에 제안된 변경사항입니다. |
| `positionedObjectIds[]` | `string`<br>이 단락에 연결된 위치가 지정된 객체의 ID입니다. |

## TextStyle

텍스트에 적용할 수 있는 스타일을 나타냅니다.

상속된 텍스트 스타일은 이 메시지에서 설정되지 않은 필드로 표시됩니다. 텍스트 스타일의 상위 요소는 텍스트 스타일이 정의된 위치에 따라 다릅니다.

- `Paragraph`의 텍스트 TextStyle은 단락의 해당 `named style type`에서 상속됩니다.
- `named style`의 TextStyle은 `normal text`라는 스타일에서 상속됩니다.
- `normal text`라는 스타일의 TextStyle은 Docs 편집기의 기본 텍스트 스타일에서 상속됩니다.
- 표에 포함된 `Paragraph` 요소의 TextStyle은 표 스타일에서 텍스트 스타일을 상속할 수 있습니다.

텍스트 스타일이 상위 요소에서 상속되지 않는 경우 필드를 설정 해제하면 스타일이 Docs 편집기의 기본값과 일치하는 값으로 되돌아갑니다.

| JSON 표현 |
| --- |
| ```<br>{<br>  "bold": boolean,<br>  "italic": boolean,<br>  "underline": boolean,<br>  "strikethrough": boolean,<br>  "smallCaps": boolean,<br>  "backgroundColor": {<br>    object (OptionalColor)<br>  },<br>  "foregroundColor": {<br>    object (OptionalColor)<br>  },<br>  "fontSize": {<br>    object (Dimension)<br>  },<br>  "weightedFontFamily": {<br>    object (WeightedFontFamily)<br>  },<br>  "baselineOffset": enum (BaselineOffset),<br>  "link": {<br>    object (Link)<br>  }<br>}<br>``` |

| 필드 |
| --- |
| `bold` | `boolean`<br>텍스트가 굵게 표시되는지 여부입니다. |
| `italic` | `boolean`<br>텍스트의 기울임체 여부입니다. |
| `underline` | `boolean`<br>텍스트에 밑줄이 그어져 있는지 여부입니다. |
| `strikethrough` | `boolean`<br>텍스트에 취소선이 그어져 있는지 여부입니다. |
| `smallCaps` | `boolean`<br>텍스트가 스몰캡인지 여부입니다. |
| `backgroundColor` | `object (OptionalColor)`<br>텍스트의 배경색입니다. 설정된 경우 색상은 `color` 필드에 따라 RGB 색상 또는 투명입니다. |
| `foregroundColor` | `object (OptionalColor)`<br>텍스트의 전경색입니다. 설정된 경우 색상은 `color` 필드에 따라 RGB 색상 또는 투명입니다. |
| `fontSize` | `object (Dimension)`<br>텍스트의 글꼴 크기입니다. |
| `weightedFontFamily` | `object (WeightedFontFamily)`<br>텍스트의 글꼴 모음 및 렌더링된 두께입니다.<br>업데이트 요청에서 `weightedFontFamily` 및 `bold`의 값을 모두 지정하면 `weightedFontFamily`가 먼저 적용된 후 `bold`가 적용됩니다.<br>`weightedFontFamily#weight`이 설정되지 않은 경우 기본값은 `400`입니다.<br>`weightedFontFamily`가 설정된 경우 `weightedFontFamily#fontFamily`도 비어 있지 않은 값으로 설정해야 합니다. 그렇지 않으면 400 잘못된 요청 오류가 반환됩니다. |
| `baselineOffset` | `enum (BaselineOffset)`<br>텍스트의 일반 위치에서 세로 오프셋입니다.<br>`SUPERSCRIPT` 또는 `SUBSCRIPT` 기준선 오프셋이 있는 텍스트는 `fontSize` 필드를 기반으로 계산된 더 작은 글꼴 크기로 자동 렌더링됩니다. 이 필드의 변경사항은 `fontSize`에 영향을 미치지 않습니다. |
| `link` | `object (Link)`<br>텍스트의 하이퍼링크 대상입니다. 설정되지 않은 경우 링크가 없습니다. 링크는 상위 텍스트에서 상속되지 않습니다.<br>업데이트 요청에서 링크를 변경하면 범위의 텍스트 스타일이 다음과 같이 변경됩니다.<br>- 링크를 설정하면 텍스트 전경색이 기본 링크 색상으로 업데이트되고 텍스트에 밑줄이 표시됩니다. 이러한 필드가 동일한 요청에서 수정되면 링크 기본값 대신 해당 값이 사용됩니다.<br>- 기존 링크와 겹치는 텍스트 범위에 링크를 설정하면 기존 링크도 새 URL을 가리키도록 업데이트됩니다.<br>- 링크는 줄 바꿈 문자에 설정할 수 없습니다. 따라서 `"ABC\n123"`와 같이 단락 경계를 교차하는 텍스트 범위에 링크를 설정하면 줄바꿈 문자가 자체 텍스트 실행으로 분리됩니다. 링크는 줄바꿈 전후의 실행에 별도로 적용됩니다.<br>- 링크를 삭제하면 동일한 요청에서 다른 스타일을 설정하지 않는 한 범위의 텍스트 스타일이 이전 텍스트의 스타일 (또는 이전 텍스트가 다른 링크인 경우 기본 텍스트 스타일)과 일치하도록 업데이트됩니다. |

## WeightedFontFamily

텍스트의 글꼴 모음과 두께를 나타냅니다.

| JSON 표현 |
| --- |
| ```<br>{<br>  "fontFamily": string,<br>  "weight": integer<br>}<br>``` |

| --- |

## ParagraphStyle

전체 단락에 적용되는 스타일입니다.

상속된 단락 스타일은 이 메시지에서 설정되지 않은 필드로 표시됩니다. 단락 스타일의 상위 요소는 단락 스타일이 정의된 위치에 따라 다릅니다.

- `Paragraph`의 ParagraphStyle은 단락의 해당 `named style type`에서 상속됩니다.
- `named style`의 ParagraphStyle은 `normal text` 명명된 스타일에서 상속됩니다.
- `normal text`라는 스타일의 ParagraphStyle은 Docs 편집기의 기본 단락 스타일에서 상속됩니다.
- 표에 포함된 `Paragraph` 요소의 ParagraphStyle은 표 스타일에서 단락 스타일을 상속할 수 있습니다.

단락 스타일이 상위 항목에서 상속되지 않는 경우 필드를 설정 해제하면 스타일이 Docs 편집기의 기본값과 일치하는 값으로 되돌아갑니다.

| JSON 표현 |
| --- |
| ```<br>{<br>  "headingId": string,<br>  "namedStyleType": enum (NamedStyleType),<br>  "alignment": enum (Alignment),<br>  "lineSpacing": number,<br>  "direction": enum (ContentDirection),<br>  "spacingMode": enum (SpacingMode),<br>  "spaceAbove": {<br>    object (Dimension)<br>  },<br>  "spaceBelow": {<br>    object (Dimension)<br>  },<br>  "borderBetween": {<br>    object (ParagraphBorder)<br>  },<br>  "borderTop": {<br>    object (ParagraphBorder)<br>  },<br>  "borderBottom": {<br>    object (ParagraphBorder)<br>  },<br>  "borderLeft": {<br>    object (ParagraphBorder)<br>  },<br>  "borderRight": {<br>    object (ParagraphBorder)<br>  },<br>  "indentFirstLine": {<br>    object (Dimension)<br>  },<br>  "indentStart": {<br>    object (Dimension)<br>  },<br>  "indentEnd": {<br>    object (Dimension)<br>  },<br>  "tabStops": [<br>    {<br>      object (TabStop)<br>    }<br>  ],<br>  "keepLinesTogether": boolean,<br>  "keepWithNext": boolean,<br>  "avoidWidowAndOrphan": boolean,<br>  "shading": {<br>    object (Shading)<br>  },<br>  "pageBreakBefore": boolean<br>}<br>``` |

| --- |

| `EVENLY_DISTRIBUTED` | 열 너비는 균등하게 분배된 다른 열 간에 균등하게 분배됩니다.<br>열의 너비는 자동으로 결정되며 지정된 너비가 있는 모든 열을 고려한 후 표에 남아 있는 너비의 동일한 부분을 갖습니다. |
| `FIXED_WIDTH` | 고정 열 너비입니다. \[너비\]\[`google.apps.docs.v1.TableColumnProperties.width` 속성에는 열의 너비가 포함됩니다. |\

## 목록\
\
목록은 모두 동일한 목록에 속하는 단락 그룹의 목록 속성을 나타냅니다. 목록의 일부인 단락에는 `bullet`에 목록의 ID에 대한 참조가 있습니다.\
\
| JSON 표현 |\
| --- |\
| ```<br>{<br>  "listProperties": {<br>    object (ListProperties)<br>  },<br>  "suggestedListPropertiesChanges": {<br>    string: {<br>      object (SuggestedListProperties)<br>    },<br>    ...<br>  },<br>  "suggestedInsertionId": string,<br>  "suggestedDeletionIds": [<br>    string<br>  ]<br>}<br>``` |\
\
| 필드 |\
| --- |\
| `listProperties` | `object (ListProperties)`<br>목록의 속성입니다. |\

| `suggestedInsertionId` | `string`<br>추천 삽입 ID입니다. 비어 있으면 추천 삽입이 아닙니다. |\
| `suggestedDeletionIds[]` | `string`<br>추천 삭제 ID입니다. 비어 있으면 이 목록의 제안된 삭제가 없습니다. |\

\
| JSON 표현 |\

\
| 필드 |\
| --- |\
| `nestingLevels[]` | `object (NestingLevel)`<br>연결된 수준의 글머리 기호 속성을 설명합니다.<br>목록에는 최대 9개의 중첩 수준이 있으며 중첩 수준 0은 최상위 수준에 해당하고 중첩 수준 8은 가장 중첩된 수준에 해당합니다. 중첩 수준은 오름차순으로 반환되며 가장 적게 중첩된 수준이 먼저 반환됩니다. |\

## NestingLevel\
\
특정 중첩 수준의 목록 글머리 기호의 디자인을 설명하는 속성을 포함합니다.\
\
| JSON 표현 |\
| --- |\
| ```<br>{<br>  "bulletAlignment": enum (BulletAlignment),<br>  "glyphFormat": string,<br>  "indentFirstLine": {<br>    object (Dimension)<br>  },<br>  "indentStart": {<br>    object (Dimension)<br>  },<br>  "textStyle": {<br>    object (TextStyle)<br>  },<br>  "startNumber": integer,<br>  // Union field glyph_kind can be only one of the following:<br>  "glyphType": enum (GlyphType),<br>  "glyphSymbol": string<br>  // End of list of possible types for union field glyph_kind.<br>}<br>``` |\
\
| 필드 |\
| --- |\
| `bulletAlignment` | `enum (BulletAlignment)`<br>글머리 기호를 렌더링하기 위해 할당된 공간 내에서 글머리 기호의 정렬입니다. |\
| `glyphFormat` | `string`<br>이 중첩 수준의 글머리 기호에서 사용되는 형식 문자열입니다.<br>글리프 형식에는 하나 이상의 자리표시자가 포함되어 있으며 이러한 자리표시자는 `glyphType` 또는 `glyphSymbol`에 따라 적절한 값으로 대체됩니다. 자리표시자는 `%[nestingLevel]` 패턴을 따릅니다. 또한 자리표시자에는 접두사와 접미사가 있을 수 있습니다. 따라서 글리프 형식은 `<prefix>%[nestingLevel]<suffix>` 패턴을 따릅니다. 접두사와 접미사는 선택사항이며 임의의 문자열일 수 있습니다.<br>예를 들어 글리프 형식 `%0.`는 렌더링된 글리프가 자리표시자를 중첩 수준 0의 해당 글리프로 바꾸고 마침표가 접미사로 추가됨을 나타냅니다. 따라서 중첩 수준 0에서 글리프 유형이 `UPPER_ALPHA`이고 글리프 형식이 `%0.`인 목록은 렌더링된 글리프가 있는 목록이 됩니다. <br>`A.`<br>`B.`<br>`C.`<br>글리프 형식에는 현재 중첩 수준의 자리표시자와 상위 중첩 수준의 자리표시자가 포함될 수 있습니다. 예를 들어 목록의 중첩 수준 0에는 `%0.` 글리프 형식이 있고 중첩 수준 1에는 `%0.%1.` 글리프 형식이 있을 수 있습니다. 두 중첩 수준 모두 `DECIMAL` 글리프 유형이 있다고 가정하면 렌더링된 글리프가 있는 목록이 생성됩니다. <br>`1.`<br>`2.`<br>`2.1.`<br>`2.2.`<br>`3.`<br>순서가 지정된 중첩 수준의 경우 특정 단락의 글리프 형식에서 자리표시자를 대체하는 문자열은 목록 내 단락의 순서에 따라 달라집니다. |\
| `indentFirstLine` | `object (Dimension)`<br>이 중첩 수준에서 단락의 첫 번째 줄에 적용할 들여쓰기 양입니다. |\
| `indentStart` | `object (Dimension)`<br>이 중첩 수준의 단락에 대한 들여쓰기 양입니다. 단락의 콘텐츠 방향에 따라 텍스트 시작에 해당하는 측면에 적용됩니다. |\
| `textStyle` | `object (TextStyle)`<br>이 중첩 수준의 글머리기호의 텍스트 스타일입니다. |\
| `startNumber` | `integer`<br>이 중첩 수준의 첫 번째 목록 항목 번호입니다.<br>문자 목록과 로마 숫자 목록의 경우 0 값은 1 값으로 처리됩니다. 값이 0과 1인 경우 문자 목록과 로마 숫자 목록은 각각 `a`와 `i`에서 시작합니다.<br>이 값은 순서가 지정되지 않은 글리프가 있는 중첩 수준에서는 무시됩니다. |\
| 통합 필드 `glyph_kind`. 글머리 기호에 사용되는 글리프의 종류입니다. `glyph_format`의 자리표시자를 대체하는 글머리 기호 글리프를 정의합니다. `glyph_kind`은 다음 중 하나일 수 있습니다. |\
| `glyphType` | `enum (GlyphType)`<br>이 수준의 중첩된 단락이 정렬될 때 글머리 기호에서 사용되는 글리프 유형입니다.<br>글리프 유형은 이 수준의 중첩에 있는 단락이 정렬될 때 `glyphFormat` 내에서 자리표시자를 대체하는 데 사용되는 글리프의 유형을 결정합니다. 예를 들어 중첩 수준이 0이고 `glyphFormat`이 `%0.`이며 글리프 유형이 `DECIMAL`인 경우 렌더링된 글리프는 글리프 형식의 자리표시자 `%0`를 목록 내 목록 항목의 순서에 해당하는 숫자로 대체합니다. |\
| `glyphSymbol` | `string`<br>이 수준의 중첩된 단락이 순서가 지정되지 않은 경우 글머리 기호에서 사용되는 맞춤 글리프 기호입니다.<br>문자 기호는 `glyphFormat` 내의 자리표시자를 대체합니다. 예를 들어 glyphSymbol이 유니코드 U+25cf 코드 포인트에 해당하는 단색 원이고 `glyphFormat`이 `%0`이면 렌더링된 글리프는 단색 원입니다. |\

## BulletAlignment\
\
글머리 기호의 정렬 유형입니다.\
\
| 열거형 |\
| --- |\
| `BULLET_ALIGNMENT_UNSPECIFIED` | 글머리 기호 정렬이 지정되지 않았습니다. |\
| `START` | 글머리 기호는 글머리 기호를 렌더링하기 위해 할당된 공간의 시작 부분에 맞춰 정렬됩니다. LTR 텍스트의 경우 왼쪽 정렬, 그 외의 경우 오른쪽 정렬 |\
| `CENTER` | 불릿은 불릿을 렌더링하기 위해 할당된 공간의 중앙에 정렬됩니다. |\
| `END` | 글머리 기호는 글머리 기호를 렌더링하기 위해 할당된 공간의 끝에 정렬됩니다. LTR 텍스트의 경우 오른쪽 정렬, 그 외의 경우 왼쪽 정렬 |\

## GlyphType\
\
이 수준의 중첩에 있는 단락이 정렬될 때 글머리 기호에서 사용되는 글리프 유형입니다.\
\
| 열거형 |\
| --- |\
| `GLYPH_TYPE_UNSPECIFIED` | 글리프 유형이 지정되지 않았거나 지원되지 않습니다. |\
| `NONE` | 빈 문자열. |\
| `DECIMAL` | `1`, `2`, `3`와 같은 숫자입니다. |\
| `ZERO_DECIMAL` | 한 자리 숫자에 0이 접두사로 붙은 숫자입니다(예: `01`, `02`, `03`). 두 자리 이상의 숫자는 0으로 시작하지 않습니다. |\
| `UPPER_ALPHA` | `A`, `B`, `C`와 같은 대문자입니다. |\
| `ALPHA` | `a`, `b`, `c`와 같은 소문자 |\
| `UPPER_ROMAN` | `I`, `II`, `III`와 같은 대문자 로마 숫자 |\
| `ROMAN` | `i`, `ii`, `iii`과 같은 소문자 로마 숫자 |\

## NamedRange\
\
이름이 지정된 범위 ID가 동일한 `Range`의 컬렉션입니다.\
\
명명된 범위를 사용하면 개발자가 문서의 일부를 임의의 사용자 정의 라벨과 연결하여 나중에 콘텐츠를 프로그래매틱 방식으로 읽거나 수정할 수 있습니다. 문서에는 이름이 같은 명명된 범위가 여러 개 포함될 수 있지만 모든 명명된 범위에는 고유 ID가 있습니다.\
\
이름이 지정된 범위는 단일 `Range`로 생성되며 이름이 지정된 범위 내에 삽입된 콘텐츠는 일반적으로 해당 범위를 확장합니다. 하지만 특정 문서 변경사항으로 인해 범위가 여러 범위로 분할될 수 있습니다.\
\
이름이 지정된 범위는 비공개가 아닙니다. 문서에 액세스할 수 있는 모든 애플리케이션과 공동작업자는 명명된 범위를 볼 수 있습니다.\
\
| JSON 표현 |\
| --- |\
| ```<br>{<br>  "namedRangeId": string,<br>  "name": string,<br>  "ranges": [<br>    {<br>      object (Range)<br>    }<br>  ]<br>}<br>``` |\
\
| 필드 |\
| --- |\
| `namedRangeId` | `string`<br>이름이 지정된 범위의 ID입니다. |\
| `name` | `string`<br>이름이 지정된 범위의 이름입니다. |\
| `ranges[]` | `object (Range)`<br>이 이름이 지정된 범위에 속하는 범위입니다. |\

## CropProperties\
\
이미지의 자르기 속성입니다.\
\
자르기 사각형은 원본 콘텐츠의 4개 모서리에서 분수 오프셋을 사용하여 표현됩니다.\
\
- 오프셋이 (0, 1) 구간에 있으면 자르기 직사각형의 해당 가장자리가 이미지의 원래 경계 직사각형 내부에 배치됩니다.\
- 오프셋이 음수이거나 1보다 큰 경우 자르기 사각형의 해당 가장자리가 이미지의 원래 경계 사각형 외부에 배치됩니다.\
- 모든 오프셋과 회전 각도가 0이면 이미지가 잘리지 않습니다.\
\
| JSON 표현 |\
| --- |\
| ```<br>{<br>  "offsetLeft": number,<br>  "offsetRight": number,<br>  "offsetTop": number,<br>  "offsetBottom": number,<br>  "angle": number<br>}<br>``` |\
\
| 필드 |\
| --- |\
| `offsetLeft` | `number`<br>오프셋은 자르기 직사각형의 왼쪽 가장자리가 원래 콘텐츠의 왼쪽 가장자리에서 얼마나 안쪽으로 떨어져 있는지를 원래 콘텐츠 너비의 비율로 지정합니다. |\
| `offsetRight` | `number`<br>오프셋은 자르기 직사각형의 오른쪽 가장자리가 원래 콘텐츠의 오른쪽 가장자리에서 얼마나 안쪽으로 떨어져 있는지를 원래 콘텐츠 너비의 비율로 지정합니다. |\
| `offsetTop` | `number`<br>오프셋은 자르기 직사각형의 상단 가장자리가 원본 콘텐츠의 상단 가장자리에서 얼마나 안쪽으로 떨어져 있는지를 원본 콘텐츠 높이의 비율로 지정합니다. |\

| `angle` | `number`<br>자르기 직사각형이 중심을 기준으로 시계 방향으로 회전한 각도(라디안)입니다. 회전은 오프셋 후에 적용됩니다. |\

## SuggestionsViewMode\
\
제안된 변경사항이 표시되는 방식을 나타내는 문서에 적용된 제안 보기 모드입니다. 모든 제안사항이 인라인으로 표시되거나, 수락되거나, 거부된 상태로 문서를 읽는 옵션을 제공합니다.\
\
| 열거형 |\
| --- |\
| `DEFAULT_FOR_CURRENT_ACCESS` | 반환된 문서에 적용된 SuggestionsViewMode는 사용자의 현재 액세스 수준에 따라 달라집니다. 사용자에게 보기 액세스 권한만 있는 경우 `PREVIEW_WITHOUT_SUGGESTIONS`이 적용됩니다. 그렇지 않으면 `SUGGESTIONS_INLINE`이 적용됩니다. 이는 기본 추천 보기 모드입니다. |\
| `SUGGESTIONS_INLINE` | 반환된 문서에 인라인으로 추천이 표시됩니다. 제안된 변경사항은 문서 내에서 기본 콘텐츠와 구분됩니다.<br>이 모드를 사용하여 문서를 가져오는 요청은 사용자에게 추천 변경사항을 볼 권한이 없는 경우 403 오류를 반환합니다. |\
| `PREVIEW_SUGGESTIONS_ACCEPTED` | 반환된 문서는 제안된 모든 변경사항이 적용된 미리보기입니다.<br>이 모드를 사용하여 문서를 가져오는 요청은 사용자에게 추천 변경사항을 볼 권한이 없는 경우 403 오류를 반환합니다. |\
| `PREVIEW_WITHOUT_SUGGESTIONS` | 문서에 제안사항이 있는 경우 반환된 문서는 모든 제안된 변경사항이 거부된 미리보기입니다. |\
