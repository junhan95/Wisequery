## 1. 개요 및 배경

### 1.1 배경
마케팅 전략 수립 및 랜딩 페이지 분석 결과, 현재 기획된 앱이 "단순 채팅 클라이언트"로 인식될 우려가 있음. **"프로젝트 관리"**, **"로컬 보안"**, **"문서 작성 워크플로우"**라는 핵심 가치를 UI/UX 및 기능 차원에서 더욱 명확하게 드러내야 함.

### 1.2 목표
1.  **정체성 강화:** 앱 실행 즉시 '채팅'이 아닌 **'프로젝트 대시보드'** 느낌을 전달.
2.  **보안 가시화:** B2B/보안 민감 타겟을 위해 **암호화 상태**를 시각적으로 확인시켜 신뢰도 확보.
3.  **워크플로우 유도:** '채팅 → 목차 → 문서'로 이어지는 흐름을 자연스럽게 유도하는 UX 장치 추가.
4.  **바이럴 요소:** 초기 마케팅('30분 챌린지')을 지원하기 위한 기능적 장치 마련.

## 2. 주요 수정 및 보완 사항 (Requirements Update)

### 2.1 UI/UX: 프로젝트 중심성 강화 (Project-First UI)

기존 기획(FR-1)은 프로젝트 생성이 기능적으로만 존재함. 이를 시각적 중심에 둡니다.

| 구분 | 변경 전 (As-Is) | 변경 후 (To-Be) | 구현 요건 |
| :--- | :--- | :--- | :--- |
| **초기 화면** | 빈 채팅 화면 또는 최근 채팅 리스트 | **프로젝트 대시보드 (Empty State 포함)** | - 앱 실행 시, 좌측 트리가 아닌 **중앙 화면에 '프로젝트 카드 뷰'** 제공.<br>- "새 프로젝트 시작하기" 버튼을 가장 크게 배치.<br>- 최근 작업 문서/채팅을 프로젝트별로 묶어서 표시. |
| **상단 헤더** | 앱 타이틀 / 설정 | **브레드크럼 (Breadcrumbs) 네비게이션** | - `WiseQuery > [프로젝트명] > [채팅/문서명]` 형태의 경로 표시.<br>- 사용자가 현재 어떤 프로젝트 공간에 있는지 항상 인지하도록 유도. |

### 2.2 보안 및 신뢰성 가시화 (Visual Security)

B2B 마케팅 전략(NFR-6 강조)에 따라, 백엔드에서만 도는 암호화 로직을 UI로 끌어올림.

| 기능 ID | 수정 내용 | 상세 기획 |
| :--- | :--- | :--- |
| **SEC-UI-01** | **API Key 보안 상태 표시기** | - 설정 화면뿐만 아니라, 메인 화면 하단(Status Bar)에 **🔒 Local Encrypted** 아이콘 및 텍스트 배치.<br>- 마우스 오버 시 툴팁: *"API Key는 AES-256으로 암호화되어 귀하의 PC에만 저장됩니다."* |
| **SEC-UI-02** | **데이터 저장 경로 명시** | - 프로젝트 생성 시, 저장될 로컬 경로(예: `C:\Users\...\Documents\WQ_Projects`)를 명시적으로 보여주고 '변경' 버튼 노출.<br>- "클라우드에 저장되지 않음" 문구 추가. |

### 2.3 워크플로우 기능 고도화 (Workflow Integration)

FR-3(제안서 작성) 기능을 마케팅 포인트인 '3단계 워크플로우'에 맞춰 UX를 다듬음.

| 기능 ID | 수정 내용 | 상세 기획 |
| :--- | :--- | :--- |
| **WF-01** | **채팅 → 문서 변환 트리거** | - 채팅창 내 AI 응답 메시지 옆에 **[📝 기획서로 변환]** 또는 **[📂 목차로 추출]** 버튼 추가.<br>- 클릭 시 즉시 '제안서 작성 탭'으로 이동하며 해당 내용이 Context로 입력됨. |
| **WF-02** | **단계별 진행 표시 (Stepper)** | - 문서 작성 화면 상단에 **[1. 기획 입력] > [2. 목차 구성] > [3. 본문 생성]** 스텝퍼(Stepper) UI 추가.<br>- 현재 단계와 다음 할 일을 명확히 안내. |

### 2.4 온보딩 및 마케팅 지원 (Onboarding & Viral)

초기 사용자 1,000명 확보 및 챌린지 이벤트를 위한 기능.

| 기능 ID | 수정 내용 | 상세 기획 |
| :--- | :--- | :--- |
| **MKT-01** | **웰컴 튜토리얼 프로젝트** | - 앱 최초 설치 시, **'WiseQuery 사용법'**이라는 예제 프로젝트가 자동 생성되어 있음.<br>- 예제 채팅과 문서가 포함되어 있어 사용자가 구조를 바로 이해하게 함. |
| **MKT-02** | **워터마크 Export 옵션** | - 문서 내보내기(PDF/Image) 시, 하단에 *"Generated by WiseQuery"* 워터마크를 넣을 수 있는 옵션 제공 (기본값: On, 해제 가능).<br>- 바이럴 마케팅 효과 증대. |
| **MKT-03** | **피드백 단축 버튼** | - 앱 상단 또는 설정 메뉴에 **"제안/버그신고"** 버튼 배치.<br>- Discord 채널 또는 구글 폼으로 바로 연결 (초기 커뮤니티 활성화 목적). |

## 3. 기술적 검토 사항 (Engineering Check)

1.  **상태 관리(Zustand) 구조 변경:**
    * 단순 채팅 리스트가 아닌, `CurrentProject` 상태를 중심으로 앱 전체 뷰가 전환되도록 스토어 구조 재점검 필요.
2.  **Tauri Window 설정:**
    * 초기 로딩 시(Splash Screen) "Initializing Local Security Module..." 등의 문구를 넣어 보안/성능 이미지를 로딩 단계부터 심어줄 것.
3.  **로컬 스토리지 접근:**
    * 튜토리얼 프로젝트 자동 생성을 위해 앱 설치 직후 `app_data_dir`에 예제 JSON 파일 복사 로직 추가 필요.

## 4. 수정된 개발 일정 반영 (Schedule Impact)

기존 P5(버그 수정 및 UX 개선) 단계의 우선순위를 조정하여 위 사항들을 반영함.

* **P3 (기획/문서 기능):** `WF-01`, `WF-02` (워크플로우 UX) 구현 포함.
* **P4 (설정/보안):** `SEC-UI-01`, `SEC-UI-02` (보안 가시화) 구현 포함.
* **P5 (UX/패키징):** `MKT-01` (웰컴 프로젝트), UI 대시보드화 작업 진행.

## 5. 결론

본 수정 기획서는 WiseQuery가 단순한 "또 하나의 GPT 래퍼(Wrapper)"가 아닌, **"대체 불가능한 로컬 프로젝트 관리 도구"**로 포지셔닝하기 위한 필수적인 조치입니다. 개발팀은 위 내용을 검토하여 v1.0 스펙에 반영해 주시기 바랍니다.
